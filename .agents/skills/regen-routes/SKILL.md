# Regenerate demo routes

The demo's member routes are real street geometry baked into `src/demo/routes.ts`,
generated by `scripts/fetch-routes.mjs` from the free OSRM demo server
(`routing.openstreetmap.de`, foot + car profiles). Members must follow actual
sidewalks/roads — never hand-draw waypoints and never re-enable Chaikin
smoothing on them (corner-cutting slices through corner buildings).

## How

1. Edit the constants at the top of `scripts/fetch-routes.mjs`:
   - `WALKS`: one `[lng, lat]` start per walking member (destination is `DEST`).
   - `DRIVES`: waypoint lists per driving corridor — include a via point to pin
     the corridor (e.g. Placerville pins US-50, Truckee pins I-80) and any POI a
     scripted stop must pass.
2. Run `node scripts/fetch-routes.mjs` (needs network). It fetches, simplifies
   (Douglas-Peucker: ~2 m walk, ~30 m drive), and rewrites `src/demo/routes.ts`.
3. Verify constraints before calling it done:
   - Scripted-stop POIs (`stopEvent.poi` in `src/demo/data.ts`) must lie within
     **300 m** of the stopping member's route (`ON_ROUTE_TOLERANCE_M` in
     `src/demo/simulation.ts`), or the pull-over never fires. Check with a quick
     min-distance calculation over the new route points.
   - `npx tsc --noEmit` passes.
4. Route keys in `routes.ts` must match `routeKey` values in `data.ts` members.

## Notes

- Coordinates are `[lat, lng]` pairs in routes.ts, converted to `LatLng` in data.ts.
- Walking routes should land ~15-25 points, drives a few hundred; if counts
  balloon, raise the simplification tolerance rather than trimming by hand.
- OSRM demo server is rate-limited but fine for this handful of requests.
